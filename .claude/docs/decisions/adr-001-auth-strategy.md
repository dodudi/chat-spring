# ADR-001: 인증 전략 — JWT 외부 인증 서버 위임

- **날짜**: 2025-05-27
- **상태**: 승인됨

## 배경

채팅 서비스 특성상 인증 자체보다 채팅 도메인에 집중하는 것이 목표였음.
직접 회원가입/로그인 API를 구현하면 users 테이블 설계, 비밀번호 해싱, 토큰 발급 등 인증 인프라 전반을 책임져야 함.

## 결정

- 인증은 **외부 JWT 발급 서버에 완전 위임**
- 이 서버는 JWT를 **검증만** 하고, 발급하지 않음
- `spring-boot-starter-security-oauth2-resource-server` 사용 (Resource Server 역할)
- **users 테이블 없음** — 사용자 식별은 JWT의 `sub` 클레임 값(userId)으로만 처리
- 로컬 개발 시에는 HMAC 시크릿 키(`app.jwt.local-secret`)로 자체 서명 토큰 검증

## 결과

- 장점: 인증 로직 부담 없음, 도메인 집중 가능
- 단점: 외부 인증 서버 없이는 로컬 테스트가 번거로움 → 로컬용 시크릿 키로 해소
- users 테이블이 없으므로 userId는 항상 JWT에서 추출한 문자열로 간주
